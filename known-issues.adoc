== VolumeSnapshots 未到达 ReadyToUse 状态

[WARNING]
====
 `csi-snapshotter` sidecar 存在已知问题。在所有 Kubernetes 版本中，VolumeGroupSnapshots `v1beta1` 阻止 VolumeSnapshots 到达 `ReadyToUse`状态。

====
有两种解决方法可供选择：

. 删除 VolumeGroupSnapshots CRD 以禁用 VolumeGroupSnapshots，然后重新安装 Trident。
. 安装 VolumeGroupSnapshots `v1beta2` 和 `snapshot-controller` 版本 8.4.0 或更高版本，然后重新安装 Trident。VolumeGroupSnapshots 在 v1.34 之前的 Kubernetes 版本上不起作用。




== 还原大型文件的Restic备份可能会失败

从使用 Restic 创建的 Amazon S3 备份中恢复 30GB 或更大的文件时，恢复操作可能会失败。作为变通方法，可以使用 Kopia 作为数据移动工具备份数据（Kopia 是备份的默认数据移动工具）。请参阅 https://docs.netapp.com/us-en/trident/trident-protect/trident-protect-protect-apps.html["使用Trident Protect 保护应用程序"^]以获取说明。
