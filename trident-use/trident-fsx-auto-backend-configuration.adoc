= 为 AWS FSx for NetApp ONTAP 配置自动后端配置
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Trident 支持 AWS FSx for NetApp ONTAP (FSxN) 的自动后端配置。

当您创建包含所需 FSxN 参数的 StorageClass 时，Trident 会自动创建相应的后端和 VolumeSnapshotClass。Trident 仅为给定的驱动程序配置创建一次 VolumeSnapshotClass，并将其重复用于后续的 StorageClasses。



== 了解自动后端配置的工作原理

Trident 从 StorageClass 定义中导出后端配置。

应用 StorageClass 以触发自动后端配置。Trident 执行以下操作：

* 验证所需参数
* 自动创建后端
* 创建 VolumeSnapshotClass（仅一次）
* 使用配置状态注释 StorageClass


配置成功后，您可以创建引用此 StorageClass 的 PersistentVolumeClaims。



== 指定所需 StorageClass 参数

在 StorageClass 的 `parameters`部分中定义所需的参数，以触发自动后端创建。

[cols="2,1,1,4"]
|===
| 参数 | Required | Type | Description 


| `fsxFilesystemID` | 是的。 | string | 用于 NetApp ONTAP 的 FSx 文件系统 ID。 


| `storageDriverName` | 是的。 | string | Trident 存储驱动程序。例如： `ontap-nas` 或  `ontap-san`。 


| `credentialsName` | 是的。 | string | 包含凭据的 AWS Secrets Manager 密钥的名称或 ARN。 
|===


== 指定可选参数

您可以通过 StorageClass 传递可选的后端参数。

FSx for NetApp ONTAP 后端支持的所有后端参数也可以直接通过 StorageClass 的 `parameters`部分进行传递。

[cols="2,1,1,4"]
|===
| 参数 | Required | Type | Description 


| 后端参数 | 可选 | string | FSx for NetApp ONTAP 支持的所有后端参数都可以直接通过 StorageClass `parameters` 部分传递。每个参数必须定义为字符串。 
|===
重要信息:

* 所有参数都必须定义为字符串。
* 所有参数都必须出现在 StorageClass 的  `parameters` 部分中。


有关支持的后端参数的完整列表，请参见：

xref:trident-use/trident-fsx-storage-backend.adoc#backend-advanced-configuration-and-examples[FSx for NetApp ONTAP 后端配置]。



== 创建 StorageClass

创建包含所需 FSxN 参数的 StorageClass。

[source, yaml]
----
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: ontap-fsx-demo
  annotations:
    description: "Demo StorageClass for FSxN"
provisioner: csi.trident.netapp.io
parameters:
  fsxFilesystemID: "fs-0abc123"
  storageDriverName: "ontap-nas"
  credentialsName: "arn:aws:secretsmanager:us-west-2:123456789012:secret:trident-creds"
allowVolumeExpansion: true
reclaimPolicy: Delete
volumeBindingMode: Immediate
----
应用 StorageClass 以触发自动后端创建。

Trident 会自动在内部创建必要的后端资源。



== 验证后端配置状态

使用 StorageClass 注释验证后端配置状态。

运行以下命令：

[source, bash]
----
kubectl get storageclass ontap-fsx-demo -o yaml
----
输出示例：

[source, yaml]
----
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  annotations:
    trident.netapp.io/additionalStoragePools: trident-nas-sc1-fs-0752884ae35c620a0-nfs:.*
    trident.netapp.io/configuratorMessage: Completed Trident backend configuration
    trident.netapp.io/configuratorName: tconf-nas-sc1
    trident.netapp.io/configuratorStatus: Success
    trident.netapp.io/managed: "true"
parameters:
  credentialsName: arn:aws:secretsmanager:us-west-2:test-arn
  fsxFilesystemID: fs-0752884ae35c620a0
  storageDriverName: ontap-nas
provisioner: csi.trident.netapp.io
reclaimPolicy: Delete
volumeBindingMode: Immediate
----
关键注释：

[cols="2,4"]
|===
| 标注 | Description 


| `trident.netapp.io/configuratorStatus` | 配置结果 (`Success` 或 `Failure`)。 


| `trident.netapp.io/configuratorMessage` | 详细的配置状态或错误消息。 


| `trident.netapp.io/configuratorName` | 内部配置器资源名称。 


| `trident.netapp.io/managed` | 表示 StorageClass 由 Trident 管理。 


| `trident.netapp.io/additionalStoragePools` | 为此后端创建的存储池。 
|===
如果配置成功(`configuratorStatus: Success`），StorageClass 则完全运行。

您现在可以：

* 使用此 StorageClass 创建 PersistentVolumeClaims
* 在应用程序工作负载中装入生成的卷




== 添加其他 FSxN 文件系统

如果在继续使用相同的 StorageClass 的同时需要额外的存储容量，请为 NetApp ONTAP 文件系统附加额外的 FSx。

编辑 StorageClass：

[source, bash]
----
kubectl edit storageclass <storageclass-name>
----
使用其他文件系统 ID 添加或更新注释：

[source, yaml]
----
metadata:
  annotations:
    trident.netapp.io/additionalFsxNFileSystemID: '["fs-03cc1a718cddd6e248"]'
----
使用其他文件系统的示例 StorageClass：

[source, yaml]
----
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: test-ontap-fsx
  annotations:
    trident.netapp.io/additionalFsxNFileSystemID: '["fs-03cc1a718cddd6e248"]'
    trident.netapp.io/additionalStoragePools: trident-test-ontap-fsx-fs-0056066a48e4b22ee91-iscsi:.*
    trident.netapp.io/configuratorMessage: Completed Trident backend configuration
    trident.netapp.io/configuratorName: tconf-test-ontap-fsx
    trident.netapp.io/configuratorStatus: Success
    trident.netapp.io/managed: "true"
parameters:
  credentialsName: arn:aws:secretsmanager:us-west-2:1866785786363:secret:trident-fsx
  fsxFilesystemID: fs-0056066a48e4b22e91
  storageDriverName: ontap-san
  svmName: testSvm
provisioner: csi.trident.netapp.io
reclaimPolicy: Delete
volumeBindingMode: Immediate
----
此操作在继续使用现有 StorageClass 的同时增加了额外的存储容量。



== 操作注意事项和限制

删除 StorageClass 通常会删除关联的 Trident 后端。

这可能会中断存储连接并中断正在运行的工作负载。

在删除任何托管 StorageClass 之前，请验证潜在影响。

只有适用于 NetApp ONTAP 的 AWS FSx 才支持自动后端配置。
